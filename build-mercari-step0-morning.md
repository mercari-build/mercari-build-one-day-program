## playground (コードを書くツール) の使い方

画面は **左右 2 つ** に分かれています。

- **左**: コードを書くところ。タブが **HTML / CSS / JS** の 3 つあります
	- 今日いじるのは **JS タブだけ** で OK (HTML/CSS はすでに整えてあります)
- **右**: 書いたコードが動いてるところ (ゲーム画面)

### 知っておくと便利

- コードを書き換えると、**約 0.6 秒後** に右側に反映されます。すぐ変わらなくても大丈夫
- **Tab キー** = 半角スペース 2 つが入ります (インデント用)
- 動かなくなったら、ブラウザを **リロード (⌘R / Ctrl+R)** すると最初の状態に戻せます

## 午前 (ハンズオン) の進め方

playground を開くと、すでにコードが書かれています。ただし **2 か所だけ穴が空いていて**、そこを埋めるとゲームが完成します。

### スターターの状態

- マウスを動かすと **かごは動きます** ✓
- でも **りんごは画面の外で止まったまま** (← 課題 ① で動かす)
- かごをぶつけても **スコアは入らない** (← 課題 ② でキャッチ判定)

### 課題 ① — りんごを落とす (`moveApples` の中を書く)

**やりたいこと**: りんごを下に落として、画面の下まで行ったら新しいりんごを上から作り直す。

考える順番:

1. `apples` 配列のりんごを全部見るために `for` ループを書く
2. ループの中で、りんごの `y` (縦位置) を `speed` ぶんだけ下げる
3. 画面の下を超えたりんごは、新しいりんごに作り替える (`makeApple()` を使う)

> 💡 詰まったら: 「とりあえず for ループの形だけ書いてみる」 → 「中で y を増やしてみる」 → 「if を足してみる」 の 3 段階で進めると見えてきます。

**👀 コードのヒント** — まず 5〜10 分は自分で考えてから、必要なものだけ開いてみよう！

<details>
<summary>for ループの書き方が分からない</summary>

```javascript
for (let i = 0; i < apples.length; i++) {
  // ここに中身を書く
}
```
</details>

<details>
<summary>りんごの y (縦位置) の増やし方</summary>

```javascript
apples[i].y += apples[i].speed;
```
</details>

<details>
<summary>画面の下まで行ったか判定する if文の書き方</summary>

```javascript
if (apples[i].y > height) {
  // ここで作り替える
}
```
</details>

<details>
<summary>新しいりんごに差し替える</summary>

```javascript
apples[i] = makeApple();
```
</details>

### 課題 ② — かごでキャッチする (`catchApples` の中を書く)

**やりたいこと**: りんごがかごに当たったらスコアを 1 増やして、そのりんごを作り直す。

考える順番:

1. ① と同じ `for` ループでりんごを 1 個ずつ見る
2. `if` で **2 つの条件** を `&&` でつなぐ:
	- りんごが画面の下まで来ている
	- りんごの x がかごの近くにある (絶対値で考えるとラク)
3. 当たっていたら、スコアを 1 増やして、そのりんごを新しく作り替える

> 💡 詰まったら: まず ① の `for` をそのままコピーして、中身だけ書き換えると形が早く決まります。

**👀 コードのヒント** — まず 5〜10 分は自分で考えてから、必要なものだけ開いてみよう！

<details>
<summary>「画面の下まで来ている」の書き方</summary>

```javascript
apples[i].y > height - 30
```
</details>

<details>
<summary>「かごの近くにある」の書き方</summary>

```javascript
abs(apples[i].x - basketX) < 55
```

`abs` は絶対値を取る関数。「かごの x との差」が 55 より小さければ近い、と判定できる。
</details>

<details>
<summary>2 つの条件を「かつ」でつなぐ書き方</summary>

```javascript
if (A && B) {
  // 両方 true のときだけ中に入る
}
```
</details>

<details>
<summary>スコアを 1 増やす書き方</summary>

```javascript
score = score + 1;
```
</details>

<details>
<summary>新しいりんごに差し替える</summary>

```javascript
apples[i] = makeApple();
```
</details>

## 困ったときの 3 ステップ

「動かない！」となったら、まず赤いエラーメッセージを確認してみましょう。

### よく出るエラー

| エラー文 | だいたいの原因 |
| --- | --- |
| `SyntaxError: Unexpected token` | 括弧 `{ } ( )` の数が合ってない、`;` の付け忘れ |
| `XXX is not defined` | 変数名のタイポ (`apples` を `apple` など)、`let` の付け忘れ |
| `Cannot read property 'y' of undefined` | 配列の範囲を超えた (`for` の条件が `<=` になってる など) |
| 画面が白いまま | `setup` `draw` のスペルミス (大文字小文字含む) |

### 元に戻したくなったら

**こまめにセーブ** しておくと安心。履歴から復元できます。

### それでもダメなときは

遠慮せずメンターに聞いてみましょう！

> メンターに聞くときのコツ: 「何をしたら」「どうなった」「どう動いてほしかった」を順に話すと、原因にたどり着きやすいです。
> 例: 「`moveApples` を書いたら、画面が白くなった。本当は赤いりんごが落ちてほしかった」

「もう少し頑張る」より、**メンターと一緒にコードを読む** 方が早いです。完成しないことより、原因不明のまま手が止まる方がもったいないので、気軽に呼んでください。
